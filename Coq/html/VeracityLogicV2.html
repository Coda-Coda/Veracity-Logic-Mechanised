Section VeracityLogic.

One of the differences to the original Veracity Logic in this current formalisation is the handling of trust.

In the original logic we have the rule:

\begin{equation*} \begin{prooftree} \AxiomC{$a^l \in A \quad kTl$} \RightLabel{ $trust\ T$} \UnaryInfC{$a^k \in A$} \end{prooftree} \end{equation*}

Instead, here the equivalent would be:

\begin{equation*} \begin{prooftree} \AxiomC{$a^l \in A \quad b^k \in T l$} \RightLabel{ $trust\ T$} \UnaryInfC{$c^k \in A$} \end{prooftree} \end{equation*}

With weights, this is:

\begin{equation*} \begin{prooftree} \AxiomC{$a^l_{w_1} \in A \quad b^k_{w_2} \in T l$} \RightLabel{ $trust\ T$} \UnaryInfC{$c^k_{w_1 \times w_2} \in A$} \end{prooftree} \end{equation*}

\(a, b, c\) lose meaning in the new approach, as each piece of evidence is directly tied to the claim it is evidence for. So, we omit \(a, b, c\) and write the following:

\begin{equation*} \begin{prooftree} \AxiomC{$l \rightsquigarrow_{w_1} A \quad k \rightsquigarrow_{w_2} T l$} \RightLabel{ $trust\ T$} \UnaryInfC{$k \rightsquigarrow_{w_1 \times w_2} A$} \end{prooftree} \end{equation*}

In Coq, we cannot use subscript notation. So for \(k \rightsquigarrow_{w_1 \times w_2} A\) we write:

k ~> A @ (w1 * w2).

Here we introduce the Coq formalisation of the Veracity Logic, still an early draft.

The key 'features' supported by this formalisation are:
  • Arbitrarily many actors and atomic claims, indexed by \(\mathbb{N}\).
  • An inductive type for claims, where claims are:
    • Basic or asserting Trust in an actor (basic claim).
    • Built from pre-existing claims by logical connectives.
      • With tagged OrL and OrR as well as a regular Or.
  • Trust handling, with trust in another actor being a basic claim.
  • Weights for all beliefs.
    • Weights are of the Q rational numbers type.
    • Every statement about a belief has a weight.
      • For now, the weights are any rational number, not restricted to \([0,1]\). This restriction could be added in the future.
  • There is also a list of basic claims which is meant to serve the same purpose as the Ps variable in the Isabelle formalisation.
    • Currently, the list holds any claims, but it could potentially be restricted to only include basic claims.

Next are the proof rules.

Believes is an inductive proposition, so the rules listed are the only way to construct evidence of the form Believes ....

In the notation P |- A ~> B @ W this means that the list of claims, P, entails that the actor A believes claim B with weight W (once the fraction is simplified fully).

The double @@ notation has the same meaning, except it skips the simplification of the fraction (assuming that has already occurred).

Reserved Notation "P |- A ~> B @ W" (at level 80).
Reserved Notation "P |- A ~> B @@ W" (at level 80).
Inductive Believes (Ps : list (actor * claim))
  : actor -> claim -> Q -> Prop :=
  | assumed (a : actor) (C : claim) (H : List.In (a, C) Ps)
     : Ps |- a ~> C @ 1
  (* bot_elim has been removed, along with Bottom. TODO: discuss this *)
  | and_intro (a : actor) (C1 C2 : claim) (w1 w2 : Q)

    (e1 : Ps |- a ~> C1 @ w1) (e2 : Ps |- a ~> C2 @ w2) 
  (*---------------------------------------------------*):
            Ps |- a ~> (C1 /\' C2) @ (Qmin w1 w2)

  | and_elim1 (a : actor) (C1 C2 : claim) (w : Q)

    (e : Ps |- a ~> (C1 /\' C2) @ w)
  (*--------------------------------*):
          Ps |- a ~> C1 @ w

  | and_elim2 (a : actor) (C1 C2 : claim) (w:Q)

    (e : Ps |- a ~> (C1 /\' C2) @ w)
  (*--------------------------------*):
          Ps |- a ~> C2 @ w

  | or_intro1 (a : actor) (C1 C2 : claim) (w : Q)

      (e : Ps |- a ~> C1 @ w)
    (*-----------------------*):
    Ps |- a ~> (C1 \/L C2) @ w

  | or_intro2 (a : actor) (C1 C2 : claim) (w : Q)

      (e : Ps |- a ~> C2 @ w)
    (*-----------------------*):
    Ps |- a ~> (C1 \/R C2) @ w

  | or_elim1 (a : actor) (C1 C2 : claim) (w : Q)

    (e : Ps |- a ~> (C1 \/L C2) @ w)
  (*--------------------------------*):
           Ps |- a ~> C1 @ w

  | or_elim2 (a : actor) (C1 C2 : claim) (w : Q)

    (e : Ps |- a ~> (C1 \/R C2) @ w)
  (*--------------------------------*):
           Ps |- a ~> C2 @ w

(* These detag rules should probably be removed, loss of information. *)
  | or_detag1 (a : actor) (C1 C2 : claim) (w : Q)

    (e : Ps |- a ~> (C1 \/L C2) @ w)
  (*--------------------------------*):
       Ps |- a ~> (C1 \/' C2) @ w

  | or_detag2 (a : actor) (C1 C2 : claim) (w : Q)

    (e : Ps |- a ~> (C1 \/R C2) @ w)
  (*--------------------------------*):
      Ps |- a ~> (C1 \/' C2) @ w

  | trust (a1 a2 : actor) (C : claim) (w1 w2 : Q)

  (e : Ps |- a2 ~> C @ w2) (eT : Ps |- a1 ~> (Basic (Trusts a2)) @ w1)
(*-------------------------------------------------------------------*):
            Ps |- a1 ~> C @ (Qred (w1*w2))
(* impl_intro, when partially applied up to just before e2,
   has a similar meaning to the Isabelle equivalent. *)
  | impl_intro (a : actor) (C1 C2 : claim) (w : Q)

     (e1 : Ps |- a ~> C1 @ 1) (e2 : Ps |- a ~> C2 @ w)
   (*-------------------------------------------------*):
            Ps |- a ~> (Implies C1 C2) @ w
  (** TODO, think about the weight for impl_intro here. *)

where "P |- A ~> B @@ W" := (Believes P A B W)
and   "P |- A ~> B @ W" := (P |- A ~> B @@ (Qred W)).

We also introduce one additional notation, for when were are working solely with weights of 1. So that we don't have to write @ 1 everywhere. This notation has an additional ~ (so, a longer squiggly arrow).

Notation "P |- A ~~> B" := (P |- A ~> B @ 1) (at level 80).
